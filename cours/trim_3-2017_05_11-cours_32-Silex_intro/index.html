Silex

http://silex.sensiolabs.org

  • Silex est un "micro-framework" PHP
  • Développé par SensioLabs
  • Repose sur certains composants du Framework Symfony
  • Beaucoup moins complexe et lourd, mais très performant
  • Courbe d'apprentissage élevée
  • Nécessite au minimum PHP 5.5

Gérer les dépendances de Silex

Composer

Composer est l'équivalent de NPM, mais pour le PHP

Nous allons donc devoir l'installer globalement sur nos ordinateurs

Mac

  • Ouvrez le terminal (dans n'importe quel dossier)
  • Exécutez les commandes suivantes

php -r "copy('https://getcomposer.org/installer', 'composer-setup.php');"


php -r "if (hash_file('SHA384', 'composer-setup.php') === '669656bab3166a7aff8a7506b8cb2d1c292f042046c5a994c43155c0be6190fa0355160742ab2e1c88d40d5be660b410') { echo 'Installer verified'; } else { echo 'Installer corrupt'; unlink('composer-setup.php'); } echo PHP_EOL;"


php composer-setup.php


sudo php -r "unlink('composer-setup.php');"


sudo mv composer.phar /usr/local/bin/composer

Window

  • Fermez le terminal
  • Télécharger et lancez l'installer
  • Rouvrez le terminal

Créer le projet SIlex

  • Créez un nouveau dossier qui contiendra notre projet
  • Ouvrez le terminal dans ce dossier
  • Lancez la commande suivante

composer require silex/silex
  • Le dossier vendor contient Silex ainsi que d'autres composants que nous verrons plus tard
  • Le fichier composer.json contient la liste des dépendances requises
  • Le fichier composer.lock contient la liste des dépendances utilisées (permet de s'assurer que tous les développeurs ont les mêmes versions)
  • Créer un dossier web/ (contiendra notre site)
  • Créer un fichier index.php dans web/
  • Créer un fichier .htaccess dans web/

.HTACCESS

Comme pour le cours de routing, le fichier .htaccess va nous permettre de rediriger toutes les requêtes vers le fichier index.php

.HTACCESS


Options -MultiViews

RewriteEngine On
RewriteBase /path/to/web
RewriteCond %{REQUEST_FILENAME} !-f
RewriteRule ^ index.php [QSA,L]

Remplacez /path/to/web par le chemin vers votre site
Exemple : Si votre chemin est http://localhost/silex/web/, indiquez /silex/web

index.php

Nous allons créer une seule route dans index.php pour commencer


<?php 

// Require dependendies
require_once __DIR__.'/../vendor/autoload.php';

// Init Silex
$app = new Silex\Application();

// Create `hello` route
$app->get('/hello', function()
{
    return 'Hello!';
});

// Run Silex
$app->run();

Nous expliquerons le code par la suite

Tester

  • Rendez-vous à l'adresse de votre site
    http://localhost/silex/web/
    Vous devriez obtenir un message d'erreur
  • Rajoutez hello à la fin de l'url
    http://localhost/silex/web/hello
    Vous devriez obtenir le message Hello!

Dans web/index.php

Autoload


require_once __DIR__.'/../vendor/autoload.php';
  • Ce code va include un script php appelé autoload
  • L'autoload permet de charger directement les scripts sans avoir à faire d'include

Instanciation de Silex


$app = new Silex\Application();
  • Ce code va instancier Silex
  • Nous allons utiliser la variable $app plusieurs fois par la suite
  • Il s'agit du cœur de notre site (ou application)
  • La syntaxe Silex\ correspond au namespace. Cela permet d'organiser ses fonctions et classes comme si elle étaient dans des dossiers pour éviter les conflits (Tuto)

Première route


$app->get('/hello', function()
{
    return 'Hello!';
});
  • Ce code va demander à Silex de créer une route accessible en tapant hello dans l'URL
  • Elle va simplement renvoyer Hello!
  • Évidemment, nous allons avoir besoin de récupérer du HTML d'ici peu

Exécuter


$app->run();
  • Lance Silex
  • run() ne doit être appelé qu'une fois

Debug

Par défaut, en cas d'erreur, Silex ne va pas renvoyer beaucoup d'informations

Il suffit de rajouter le code suivant après l'avoir instancié pour obtenir plus de détails



$app = new Silex\Application();
$app['debug'] = true;
